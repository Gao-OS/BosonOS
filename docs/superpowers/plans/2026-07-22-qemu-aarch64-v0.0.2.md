# BosonOS QEMU AArch64 v0.0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a native, boot-tested `qemu-aarch64` BosonOS target and publish it with the existing x86_64 target in `v0.0.2`.

**Architecture:** The flake exposes identical logical outputs for `x86_64-linux` and `aarch64-linux`, with each native package set assembling its own kernel, Gluon, BusyBox, OTP release, rootfs, image, and QEMU app. Architecture-specific QEMU and kernel data lives in target definitions; GitHub Actions maps each release device ID to a native runner and verifies both released artifacts through serial-console E2E boots.

**Tech Stack:** Nix flakes, Linux, QEMU TCG, Zig 0.15, Elixir 1.18, OTP 27, GitHub Actions, UTM QEMU backend.

### Task 1: Add The Native AArch64 Flake Target

**Files:**
- Create: `nix/targets/qemu/aarch64.nix`
- Create: `nix/images/qemu-aarch64.nix`
- Modify: `flake.nix`

- [ ] **Step 1: Verify the AArch64 output is absent**

Run:

```bash
nix eval --json .#packages.aarch64-linux.boson-system.meta.targetName
```

Expected: FAIL because `packages.aarch64-linux` does not exist.

- [ ] **Step 2: Define the AArch64 target**

Create `nix/targets/qemu/aarch64.nix`:

```nix
{ lib }:

{
  name = "qemu-aarch64";
  soc = "qemu-virt";
  board = "qemu-aarch64";
  arch = "aarch64-linux";
  cpu = "aarch64";
  kernelTarget = "Image";
  bootloader = "qemu-direct";
  firmwareFamily = "qemu";
  defaultConsole = "ttyAMA0,115200";
  qemuBinary = "qemu-system-aarch64";
  qemuMachine = "virt,accel=tcg";
  qemuCpu = "max";
  commonCmdline = [
    "console=ttyAMA0,115200"
    "panic=5"
  ];
}
```

- [ ] **Step 3: Define the AArch64 image entrypoint**

Create `nix/images/qemu-aarch64.nix` with the same `mkImage` contract as the
x86 image and the name `boson-qemu-aarch64-image`.

- [ ] **Step 4: Make flake scope architecture-aware**

Set:

```nix
supportedSystems = [
  "x86_64-linux"
  "aarch64-linux"
];
```

Inside `systemScope`, select target and image module by system:

```nix
targetModule =
  if system == "x86_64-linux" then
    ./nix/targets/qemu/x86_64.nix
  else
    ./nix/targets/qemu/aarch64.nix;
imageModule =
  if system == "x86_64-linux" then
    ./nix/images/qemu-x86_64.nix
  else
    ./nix/images/qemu-aarch64.nix;

target = import targetModule { inherit (pkgs) lib; };
```

Call `imageModule` for `qemuImage`. Preserve the logical package names so
native runners can continue using `.#boson-qemu-image`.

- [ ] **Step 5: Verify both systems evaluate**

Run:

```bash
nix eval --json .#packages.x86_64-linux.boson-system.meta.targetName
nix eval --json .#packages.aarch64-linux.boson-system.meta.targetName
nix flake check --no-build
```

Expected: the first two commands print `qemu-x86_64` and `qemu-aarch64`; flake
evaluation passes while omitting non-native builds.

- [ ] **Step 6: Commit the target graph**

```bash
git add flake.nix nix/targets/qemu/aarch64.nix nix/images/qemu-aarch64.nix
git commit -m "feat(nix): add native qemu aarch64 target"
```
